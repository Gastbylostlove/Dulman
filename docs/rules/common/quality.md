# 로직 품질

## 추론 행동 확인

- 체크리스트나 절차에 명시된 행동은 그대로 수행한다.
- 명시되지 않은 행동을 추론했다면 실행 전 근거를 확인한다.
- 요구사항 밖의 개선은 제안으로 남기고 임의로 구현하지 않는다.

## 코드 작성 전 검증

### 프로세스

1. **사용 패턴 확인**: 대상 파일이 관련 모듈을 어떻게 import하고 호출하는지 확인한다.
2. **맥락 파악**: 입력, 처리, 출력, 부작용의 흐름을 추적한다.
3. **기존 기능 탐색**: 같은 기능이나 유틸이 이미 있는지 검색한다.
4. **구현**: 기존 패턴과 가장 가까운 방식으로 최소 변경한다.
5. **검증**: 작성한 코드가 의도와 계약에 맞는지 확인한다.

### 마이그레이션/리팩토링 시 필수

- [ ] 같은 기능을 사용하는 파일을 여러 개 확인한다.
- [ ] import 경로와 public export를 확인한다.
- [ ] rename, signature 변경, 타입 변경 후 전체 참조를 검색한다.
- [ ] 변경한 패턴으로 다시 검색해 누락을 확인한다.

## 코드 분석

### 원칙

- 코드 근거 없이 결론을 내리지 않는다.
- 입력에서 출력까지 실제 실행 순서대로 추적한다.
- 발견한 문제가 실제로 발생 가능한 시나리오인지 반증한다.

### 검색 범위

- 요청된 파일이나 폴더뿐 아니라 관련 공통 모듈, 타입, 테스트, 호출부를 함께 검색한다.
- "이미 존재하는 기능일 수 있다"는 가설을 먼저 검증한다.
- 새 코드가 기존 기능을 중복 구현하지 않는지 확인한다.

### 엣지 케이스 질문

- 이 값이 `null` 또는 `undefined`면 어떻게 되는가?
- 빈 배열, 빈 문자열, 0 같은 빈 값은 어떻게 처리되는가?
- 초기화 전에 호출되거나 여러 번 호출되면 어떻게 되는가?
- 동시에 실행되면 순서, 잠금, 중복 처리 문제가 생기는가?
- 현재 런타임에서 사용할 수 없는 API를 직접 참조하는가?
- 권한, 인증, 만료, 타임아웃, 재시도 정책은 기존 방식과 일치하는가?

### 일관성 확인

- 같은 모듈의 에러 처리 방식과 일치하는가?
- 저장과 로드의 직렬화/역직렬화가 짝을 이루는가?
- 외부 계약이 되는 문자열, 타입, 상태값을 임의로 바꾸지 않았는가?
- 환경별 분기나 설정이 기존 위치와 같은 책임 경계에 있는가?

### 분석 결과 검증 (보고 전 필수)
- 실행 환경의 특성(동시성, 메모리 모델, 실행 순서)을 이해했는가?
- 코드를 머릿속으로 실제 실행해봤는가?
- "이게 실제로 문제가 되는 시나리오가 있나?" 반증해봤는가?
- 양을 채우려고 없는 문제를 만들어낸 건 아닌가?

## 셀프 리뷰

코드 작성 후 변경한 블록마다 아래 질문을 통과해야 한다.

1. **이 코드가 없으면 무엇이 깨지는가?** 답하지 못하면 제거한다.
2. **참조 코드 대비 추가된 동작이 있는가?** 근거가 없으면 제거한다.
3. **더 단순하게 쓸 수 있는가?** 가능하면 단순화한다.
4. **요청에 없는 것을 넣었는가?** 있으면 제거하거나 사용자 확인을 받는다.

## 실무 체크

- [ ] 타입 에러, import 오류, lint 오류가 없는지 확인한다.
- [ ] 변경한 함수, 타입, 파일명의 모든 참조가 업데이트되었는지 확인한다.
- [ ] public API 문서나 주석이 실제 코드와 일치하는지 확인한다.
- [ ] 관련 테스트, 타입 검사, 빌드 중 가장 좁은 검증을 실행한다.

## 실패 패턴 기록

반복되는 실수는 아래 형식으로 기록한다.

```markdown
### [패턴명]

- **뭘 했나**: ...
- **왜 문제인가**: ...
- **어떻게 해야 했나**: ...
```

### 솔루션 퍼스트

- **뭘 했나**: 답을 먼저 정하고 대상 파일의 기존 사용 패턴을 확인하지 않았다.
- **왜 문제인가**: 기존 설계 의도와 public API를 무시한 변경이 된다.
- **어떻게 해야 했나**: 대상 파일의 import, 호출부, 비슷한 구현을 먼저 확인한 뒤 같은 방식으로 제안한다.

### 절차 생략

- **뭘 했나**: 결과가 같다고 판단해 백업, 승인, 검증, 기록 같은 중간 단계를 건너뛰었다.
- **왜 문제인가**: 절차의 각 단계는 독립적인 품질 게이트다.
- **어떻게 해야 했나**: 절차를 순서대로 실행하고, 생략이 필요하면 먼저 근거를 남긴다.
