# 핵심 실행 원칙

이 문서는 역할과 작업 유형에 무관하게 모든 Avatye AI 에이전트가 항상 적용하는 실행 원칙이다.
코드 구현 세부 원칙은 `.claude/rules/coding/principles.md`, 계약/경계 판단은 `.claude/rules/common/boundaries.md`를 따른다.

## 실행 원칙

- **생각 먼저**: 가정, 불확실성, 트레이드오프를 명시한다. 의미가 갈리는 요구사항은 구현 전에 확인한다.
- **단순함 최우선**: 요청받지 않은 기능, 설정, 확장성, 추상화를 추가하지 않는다.
- **외과적 변경**: 요청과 직접 연결된 줄만 바꾼다. 인접 코드 개선, 포맷 변경, unrelated cleanup을 섞지 않는다.
- **목표 주도**: 성공 기준을 먼저 정하고, 완료 전 가장 좁은 테스트나 검증으로 확인한다.
- **기존 자산 우선**: 프로젝트의 기존 추상화, 공통 모듈, 응답 형식, 에러 처리, 로깅, 캐시, DB 접근 패턴을 확인하고 재구현하지 않는다.
